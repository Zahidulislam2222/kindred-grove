## 1. Getting oriented

**Where to edit what:**

| Task | Where |
|---|---|
| Add a product | Admin → Products → Add product |
| Write copy on the homepage | Online Store → Themes → Customize → select *index* |
| Change brand colors / fonts | Customize → Theme settings (gear icon, bottom of left panel) |
| Add a farm to the origin pages | Content → Metaobjects → Farm → Add entry |
| Edit a page's text | Online Store → Pages → select page → Edit |
| Add a language | Settings → Languages → Add language |
| Change the store password | Online Store → Preferences → Password protection |
| View order analytics | Analytics dashboard |

Everything visible on the storefront is merchant-editable through one of these four panels: **Customize** (the theme editor), **Products**, **Content/Metaobjects**, and **Pages**.

## 2. Theme customizer basics

The theme editor is **Online Store → Themes → Customize**.

### Anatomy

- **Top bar** — switches between templates (index, product, collection, etc.)
- **Left panel** — section/block tree for the current template
- **Center** — live preview
- **Right panel** — settings for the currently selected section or block
- **Gear icon (bottom of left panel)** — theme-wide settings (brand colors, typography, social links, etc.)

### Common patterns

- **Add a block:** click the "+ Add block" button inside a section's tree.
- **Reorder blocks:** drag them up/down in the left panel.
- **Remove a block:** hover the block in the left panel → click the eye to hide, or the trash to remove.
- **Revert a change:** Save publishes. Before saving, you can Discard. After saving, use Theme library → Actions → Older versions to restore.

### Presets

Every section ships with sensible "presets" — pre-composed starting points. Use them when adding new pages instead of building from scratch:

- Homepage → "Default Kindred Grove homepage"
- Wholesale → "Wholesale page"
- Origin → "Farm detail"
- Quiz → comes with default questions + personas

## 3. Editing the homepage

Template: `index`. Default sections, top to bottom:

1. **Announcement bar** — editable top strip (e.g. "Free shipping on orders over $60").
2. **Hero** — background image + eyebrow + headline + body + CTAs. Supports `Small`, `Medium`, `Tall` heights; light or dark text theme.
3. **Values strip** — 3–5 icons + labels (e.g. Halal-certified, Single-origin, Fair-trade).
4. **Featured collections** — card grid linking to flagship collections.
5. **Farm story** — metaobject-driven. Pick a Farm record; the block pulls name, region, photo, story.
6. **Reviews carousel** — up to 8 reviews with star rating, name, product linked.
7. **Recipe suggestions** — metaobject-driven recipe cards.
8. **Recommendations** — auto-generated by Shopify's recommendations API (related or complementary).
9. **Newsletter** — email capture (Klaviyo if configured, /contact fallback otherwise).
10. **Instagram strip** — up to 8 Instagram image links.

### Common homepage tweaks

- **Change the hero image or headline:** click Hero in the left panel → update Image / Headline / Body / CTA in the right panel.
- **Swap a featured collection:** inside Featured collections → select a card → change the Collection picker.
- **Hide a section temporarily:** click the eye icon next to the section name.

## 4. Product pages

Template: `product` (applies to every product unless you pick a custom template).

### Structure

PDPs are composed of blocks — you can reorder, add, or remove them per-template. Default order:

- Breadcrumbs
- Gallery (with 3D model-viewer support when a model is attached)
- Product info:
  - Title, price, description
  - Variant picker
  - Add-to-cart
  - Trust badges
- Farm story (metaobject)
- Certifications (metaobject)
- Recipe suggestions (metaobject)
- Product FAQ
- Recommendations

### Adding product content

| Field | Where | Renders as |
|---|---|---|
| Title, description, price | Admin → Product | Product card + PDP header |
| Images, video | Admin → Product → Media | Gallery |
| Variants (size, flavor) | Admin → Product → Variants | Variant picker |
| Farm | Admin → Product → Metafields → `custom.farm` | Farm story block |
| Certifications | Admin → Product → Metafields → `custom.certifications` | Certification badges |
| Tasting notes | Admin → Product → Metafields → `custom.tasting_notes` | Under the price |
| Harvest date | Admin → Product → Metafields → `custom.harvest_date` | Product details |
| Processing method | Admin → Product → Metafields → `custom.processing_method` | Product details |
| Recipe suggestions | Admin → Product → Metafields → `custom.recipe_suggestions` | Recipe block |

Product metafields are configured once in Settings → Custom data → Products → Add definition (see `docs/metaobjects/SCHEMAS.md` for the keys).

## 5. Collections

Template: `collection.json`. Every collection page uses the same template by default.

### Structure

- Optional collection hero (when `collection.image` is set)
- Filters (left side) — auto-pulled from Shopify's storefront filters (type, vendor, availability, price, tags, metafields). Merchant-editable order from the theme editor.
- Sort dropdown
- Product grid (4-up desktop, 2-up mobile)
- Pagination or "Load more"

### Adjusting filters

- Shopify admin → Search & Discovery → Filters. Add, remove, or reorder.
- The theme reads whatever you configure there; no code change needed.

## 6. Cart + checkout

**Cart drawer** — slides in from the right when someone clicks Add-to-cart. Editable blocks:

- Free-shipping progress bar (threshold set in the block setting, in cents)
- Line item list
- Cart upsell — pick a product to promote when the cart is non-empty and that product isn't already in it
- Gift note — optional 500-char textarea

**Cart page** (`/cart`) — server-rendered, works with or without JavaScript.

**Checkout UI extension** — the trust-badge block (lives in `extensions/checkout-trust-badges/`) is scaffolded. To deploy it into checkout, a developer runs `shopify app deploy` from a separate app project — this is a one-time dev task.

## 7. Wholesale page

Create at Admin → Pages → Add page. Choose template `page.wholesale`. That's it — the wholesale form renders with a sensible default hero + intro copy.

**Form submissions** land in your Shopify admin under Customers → with the `wholesale-inquiry` tag. Filter the customer list by that tag to see new leads.

**Optional draft-order automation:** if a developer has deployed the Cloudflare Worker at `scripts/wholesale-draft-order-worker.js`, paste the Worker URL into the block setting. Every submission then also creates a draft order in Shopify Admin → Draft Orders with the company + volume + message in the notes. Without the Worker URL, submissions go to /contact only.

**Bot protection:**
- Hidden honeypot field silently drops naive bots.
- 8-second client rate-limit prevents accidental double-submits.
- The Worker (when deployed) caps payload size at 16 KB and sanitizes fields.

## 8. Origin / farm pages

One origin page per source farm. Process:

1. Admin → Content → Metaobjects → Farm → Add entry. Fill name, country, region (optional), story, photo, farmer, established year, harvest season.
2. Admin → Pages → Add page. Template: `page.origin`. Title: the farm's name.
3. In the page's metafield panel, link the page to the Farm metaobject you just made (field `custom.farm`).
4. Optionally, populate `custom.linked_products` with the products sourced from that farm — they render as a 4-up grid at the bottom.

The Origin page will show:
- Breadcrumb (Home → Origins → this page)
- Farm photo + headline + meta strip (country, region, established, harvest season)
- Full farm story (rich text from the metaobject)
- Farmer card (if linked)
- Products from that farm (if linked)

## 9. Build Your Pantry quiz

Lives at `/pages/quiz` (template `page.quiz`). The quiz is merchant-editable — you don't need a developer to change questions.

### Editing questions

1. Customize → navigate to the quiz page → select the Pantry quiz block.
2. In the right panel, find the `questions_json` and `personas_json` textareas.
3. Edit the JSON. Schema:

```json
{
  "questions": [
    {
      "text": "How do you usually cook?",
      "options": ["Weeknight fast", "Slow Sunday projects", "Eat out mostly", "Meal-prep on weekends"]
    }
  ],
  "personas": [
    {
      "name": "The Weeknight Chef",
      "blurb": "Quick, flavorful pantry picks.",
      "collection_handle": "weeknight-essentials"
    }
  ]
}
```

4. Save. The quiz re-renders with your new content on the next page load.

### How scoring works

Each answer has an index (0 to N-1). The quiz sums indexes across questions, divides by the max possible score into quartiles, and picks the matching persona. Order your personas from "low index" (fastest / simplest) to "high index" (most involved) for intuitive results.

### Result → shopping

Each persona is linked to a collection handle. After the quiz, the CTA is "Shop your pantry" → that collection. Make sure the collection handles in the JSON match real collections in your catalog.

## 10. Metaobjects

Central content records that multiple products/pages can reference. Defined in **Settings → Custom data → Metaobjects**. Full schema: `docs/metaobjects/SCHEMAS.md`.

| Type | Use case |
|---|---|
| `farm` | One record per physical farm you source from |
| `farmer` | Optional — individual farmer bio + photo |
| `region` | Optional — grouping (e.g. "Galilee") |
| `certification` | Halal, Organic, Fair Trade — one record per body |
| `recipe` | Recipe content linkable from products |

### Typical workflow

1. Create metaobject definitions once per store (schema is in `SCHEMAS.md`).
2. Add individual records as you onboard farms, get certifications, publish recipes.
3. Link them from products (via product metafields) or pages (via page metafields).
4. Blocks on the storefront auto-read the linked records and render them.

If a block is configured to read a metaobject but the record is missing, the block falls back to its own editable settings — so you can ship a page before populating the metaobject if you need to.

## 11. Predictive search

Appears in the header search input + the `/search` page. Types 2+ characters → dropdown shows matching products, pages, and blog posts in real time.

**Merchant tuning:**
- Search synonyms, featured products, search rules → Admin → Search & Discovery → Search.
- Turning off a product from search → Product → Search engine listing → uncheck "Available".
- Result limit per type: Customize → header → predictive-search block settings.

No keyword-density / SEO magic to worry about inside the theme — that's the Search & Discovery app's territory.

## 12. Newsletter + Klaviyo

The newsletter block captures email in two modes:

**Mode A — Shopify contact form only** (default, zero setup): Signup tags the subscriber as `newsletter` in Customers. Export that tag into any email tool.

**Mode B — Direct to Klaviyo** (recommended for real campaigns):

1. Sign up at klaviyo.com.
2. Create a list. Note the list ID.
3. Get your public API key from Klaviyo → Account → Settings → API Keys.
4. In Shopify theme editor → Theme settings → Observability → Klaviyo → paste the public API key.
5. In Customize → the Newsletter block → paste the list ID.
6. Done. New signups now land directly in Klaviyo with the source field "Kindred Grove storefront".

Fallback safety: even if the Klaviyo call fails (rate-limited, service down), the form falls through to the Shopify /contact endpoint so no signups are lost.

## 13. Observability

**Sentry — error tracking:**
1. Create a Sentry project (sentry.io → Create project → Browser).
2. Copy the DSN.
3. Customize → Theme settings → Observability → Error tracking → paste DSN.
4. Save.

The moment a DSN is set, Sentry starts catching JS errors from every visitor. PII is scrubbed by default (no emails, phones, addresses in reports). View errors at sentry.io in your project dashboard.

**Google Analytics 4:**
- Admin → Sales channels → Online Store → Preferences → Google Analytics → connect account.
- Core Web Vitals ship automatically as custom events (`web_vitals` → metric name + value).
- Sampling rate is merchant-configurable: Customize → Theme settings → Observability → RUM sampling rate. Default 100%; drop to 20% on production once traffic is meaningful.

## 14. Languages + currencies

**Enable a new language:**
1. Admin → Settings → Languages → Add language.
2. Pick (e.g.) Arabic. Set published.
3. The theme auto-switches via the header language switcher. Arabic automatically gets RTL layout because the theme reads `request.locale.direction`.
4. Translations live in `locales/<code>.json` (already shipped: `en.default.json`, `ar.json`).

**Enable additional currencies:**
1. Admin → Settings → Markets → edit the market → Currencies.
2. Enable USD, EUR, GBP (or any mix).
3. The header currency switcher auto-populates.

## 15. Feature flags

For lightweight A/B tests and dark launches. Configured at Customize → Theme settings → Feature flags → Flag definitions.

Format (JSON):

```json
{
  "pantry_quiz": { "default": true },
  "hero_copy": { "default": "variant_a", "variants": ["variant_a", "variant_b"], "rollout": 50 }
}
```

Blocks check flags like:

```js
if (window.KG_FF.isEnabled('pantry_quiz')) { ... }
if (window.KG_FF.variant('hero_copy') === 'variant_b') { ... }
```

**URL override** — append `?kg_ff=pantry_quiz,true` to any URL to force a flag state for QA. The override persists in localStorage for the session.

See `docs/adr/008-localstorage-feature-flags.md` for the full threat model — **never gate pricing or checkout content** with flags.

## 16. FAQs / common tasks

**Q: How do I update the logo?**
Customize → Theme settings (gear icon) → Brand → Logo.

**Q: How do I change the primary color (olive)?**
Customize → Theme settings → Brand — Colors → Primary. Color picker. Save.

**Q: How do I add a new block to the homepage?**
Customize → Index template → scroll the left panel → "+ Add block" inside the section you want. Pick the block. Drag to reorder.

**Q: How do I preview a change without publishing?**
Every customize edit is in draft until you click Save. To test on a different theme without touching the live one: Themes library → Customize on an unpublished theme.

**Q: A block doesn't show up on the page. Why?**
Three common reasons:
1. Block is hidden — check the eye icon in the left panel.
2. Block is gated by a feature flag (e.g. the quiz). Check Theme settings → Feature flags.
3. Block depends on a metaobject or metafield you haven't populated. Open the block's settings panel — the info text explains the data dependency.

**Q: How do I add a new farm?**
Content → Metaobjects → Farm → Add entry. Fill the fields. Save. Link it from any product's `custom.farm` metafield or any page's `custom.farm` metafield (for origin pages).

**Q: A customer reports a bug — where do I look first?**
Check Sentry dashboard (errors with stack traces), then the browser console while reproducing. File a detailed report including: page URL, browser, OS, what they clicked, what happened vs. expected.

**Q: Where's the changelog for this theme?**
`docs/CHANGELOG.md` — every version-tagged release documents what shipped.

**Q: Something is broken after I made a change. Rollback?**
Online Store → Themes → Theme library → three-dot menu on the current theme → Older versions → Preview → Publish.
